Módulo 09: Pipeline y Filtrado

¿Qué es el Pipeline?

El pipeline (tubería) es una de las características más poderosas de PowerShell. Permite pasar la salida de un comando como entrada al siguiente comando, encadenando múltiples operaciones.

# Pipeline básico Get-Process | Select-Object -First 5 # Pipeline múltiple Get-Service | Where-Object {$_.Status -eq "Running"} | Sort-Object Name
Diferencia clave: A diferencia de CMD que pasa texto, PowerShell pasa objetos .NET completos a través del pipeline.

Where-Object (Filtrado)

Where-Object filtra objetos en el pipeline basándose en condiciones.

Sintaxis básica

# Filtrar procesos por nombre Get-Process | Where-Object {$_.ProcessName -like "*chrome*"} # Filtrar servicios en estado específico Get-Service | Where-Object {$_.Status -eq "Stopped"} # Filtrar archivos por tamaño Get-ChildItem | Where-Object {$_.Length -gt 1MB} # Usando $_ para referenciar el objeto actual Get-Process | Where-Object {$_ CPU -gt 100}

Operadores de filtrado

# Igualdad Get-Service | Where-Object {$_.Status -eq "Running"} # Contiene Get-Process | Where-Object {$_.ProcessName -like "*word*"} # Comienza con Get-Process | Where-Object {$_.ProcessName -like "svch*"} # Termina con Get-ChildItem | Where-Object {$_.Name -like "*.txt"} # Mayor/Menor que Get-ChildItem | Where-Object {$_.Length -gt 500KB} Get-ChildItem | Where-Object {$_.Length -lt 100B}

Filtrado abreviado

# Sintaxis abreviada de Where-Object Get-Process | Where-Object CPU -gt 100 # Múltiples condiciones Get-Service | Where-Object {$_.Status -eq "Running" -and $_.StartType -eq "Automatic"} # No igual Get-Process | Where-Object {$_.ProcessName -ne "explorer"}

Select-Object (Selección)

Select-Object selecciona propiedades específicas de los objetos.

# Seleccionar propiedades específicas Get-Process | Select-Object ProcessName, Id, CPU # Seleccionar primeros/últimos elementos Get-Process | Select-Object -First 10 Get-Service | Select-Object -Last 5 # Seleccionar elementos únicos Get-Process | Select-Object -Unique ProcessName # Crear propiedades calculadas Get-Process | Select-Object ProcessName, @{Name="MemoriaMB";Expression={[math]::Round($_.WorkingSet/1MB,2)}} # Excluir propiedades Get-Process | Select-Object * -ExcludeProperty HandleCount

Propiedades calculadas

# Propiedad calculada simple Get-ChildItem | Select-Object Name, @{Name="SizeMB";Expression={[math]::Round($_.Length/1MB,2)}} # Propiedad con expresión condicional Get-Service | Select-Object Name, Status, @{Name="Estado";Expression={ if ($_.Status -eq "Running") {"Activo"} else {"Inactivo"} }}

Sort-Object (Ordenamiento)

# Ordenar por una propiedad Get-Process | Sort-Object CPU # Ordenar descendente Get-Process | Sort-Object CPU -Descending # Ordenar por múltiples propiedades Get-Process | Sort-Object CPU -Descending, Name # Ordenar archivos por fecha Get-ChildItem | Sort-Object LastWriteTime -Descending # Ordenar sin distinguir mayúsculas Get-Process | Sort-Object ProcessName -CaseSensitive

Measure-Object

Measure-Object calcula medidas estadísticas de los objetos.

# Contar objetos Get-Process | Measure-Object (Get-Process | Measure-Object).Count # Sumar valores Get-ChildItem | Measure-Object -Property Length -Sum (Get-ChildItem | Measure-Object -Property Length).Sum # Estadísticas completas Get-ChildItem | Measure-Object -Property Length -Average -Sum -Maximum -Minimum # Contar líneas en archivos Get-ChildItem -Filter "*.ps1" | Measure-Object -Property LineCount -Sum

Group-Object

Group-Object agrupa objetos por una propiedad.

# Agrupar procesos por nombre Get-Process | Group-Object ProcessName # Agrupar archivos por extensión Get-ChildItem | Group-Object Extension # Contar elementos por grupo Get-Process | Group-Object ProcessName | Where-Object {$_.Count -gt 1} # Agrupar servicios por estado Get-Service | Group-Object Status # Mostrar solo nombre del grupo y cuenta Get-ChildItem | Group-Object Extension | Select-Object Name, Count

ForEach-Object

ForEach-Object ejecuta un script block para cada objeto en el pipeline.

# Procesar cada elemento 1, 2, 3, 4, 5 | ForEach-Object { $_ * 2 } # Usando $_ para referenciar el objeto actual Get-Process | ForEach-Object { Write-Host "Proceso: $($_.ProcessName)" } # Usando el alias % Get-Service | Where-Object {$_.Status -eq "Running"} | % { $_.Name } # Múltiples comandos en el script block Get-ChildItem | ForEach-Object { Write-Host "Archivo: $($_.Name)" Write-Host "Tamaño: $($_.Length) bytes" Write-Host "---" }

ForEach-Object con begin, process, end

1..5 | ForEach-Object -Begin { $contador = 0 Write-Host "Comenzando..." } -Process { $contador++ Write-Host "Valor: $_, Iteración: $contador" } -End { Write-Host "Total procesados: $contador" }

Realizaciones Prácticas

Ejemplo 1: Reporte de procesos

Get-Process | Where-Object {$_.CPU -gt 100} | Sort-Object CPU -Descending | Select-Object -First 10 ProcessName, Id, CPU, WorkingSet

Ejemplo 2: Archivos grandes

Get-ChildItem -Recurse -File | Where-Object {$_.Length -gt 100MB} | Sort-Object Length -Descending | Select-Object FullName, @{Name="SizeGB";Expression={[math]::Round($_.Length/1GB,2)}} | Select-Object -First 20

Ejemplo 3: Servicios automáticos

Get-Service | Where-Object {$_.StartType -eq "Automatic" -and $_.Status -eq "Stopped"} | Select-Object Name, DisplayName | Sort-Object Name

Ejemplo 4: Estadísticas de archivos

Get-ChildItem | Group-Object Extension | Select-Object Name, @{Name="TotalSize";Expression={[math]::Round(($_.Group | Measure-Object -Property Length -Sum).Sum/1MB,2)}}, Count | Sort-Object TotalSize -Descending

Ejercicio Práctico

Práctica: Dominando el Pipeline
  1. Obtén los 5 procesos que más CPU están usando
  2. Lista todos los archivos .txt en tu carpeta de documentos ordenados por tamaño
  3. Cuenta cuántos servicios están en estado "Stopped"
  4. Genera un reporte de archivos agrupados por extensión con su tamaño total
  5. Encuentra todos los procesos que comiencen con "svch" y muestra su memoria
  6. Usa Measure-Object para calcular el promedio de CPU de todos los procesos
  7. Lista los 10 archivos más grandes en tu sistema (requiere permisos)
  8. Crea un pipeline que filtre, ordene y seleccione propiedades de Get-Service

Mejores Prácticas

  • Ordena los comandos eficientemente: Filtra primero, luego selecciona
  • Usa Where-Object antes de Select-Object: Reduce la cantidad de datos procesados
  • Aprovecha las abreviaturas: % para ForEach-Object, ? para Where-Object
  • Usa propiedades calculadas: Para mostrar datos formateados
  • Evita pipelines innecesarios: Si puedes usar parámetros, mejor
