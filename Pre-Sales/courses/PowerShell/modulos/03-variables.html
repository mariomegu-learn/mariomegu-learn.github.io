Módulo 03: Variables y Tipos de Datos

¿Qué son las Variables?

Las variables son contenedores que almacenan datos en memoria. En PowerShell, las variables comienzan con el símbolo $ y son case-insensitive (no distinguen mayúsculas/minúsculas).

# Crear una variable $nombre = "Carlos" $edad = 30 $activo = $true # Usar la variable Write-Host "Nombre: $nombre" Write-Host "Edad: $edad"
Consejo: PowerShell es case-insensitive: $Nombre, $NOMBRE y $nombre son la misma variable.

Tipos de Datos Básicos

1. String (Cadena de texto)

$texto = "Hola Mundo" $comillas = 'Él dijo "Hola"' $multilinea = @" Esta es una línea Esta es otra línea "@

2. Integer (Entero)

$numero = 42 $grande = 1000000 $negativo = -50

3. Double/Decimal (Punto flotante)

$pi = 3.1416 $precio = 19.99 $decimal = [decimal]10.50

4. Boolean (Verdadero/Falso)

$verdadero = $true $falso = $false $activo = $null

5. Array (Arreglo)

# Crear arrays $numeros = 1, 2, 3, 4, 5 $nombres = @("Ana", "Carlos", "Beatriz") $mixto = 1, "texto", $true, 3.14 # Array vacío $vacio = @()

6. Hashtable (Tabla hash)

# Crear hashtable $persona = @{ Nombre = "Carlos" Edad = 30 Ciudad = "Madrid" } # Acceder a valores $persona["Nombre"] $persona.Nombre

Declaración de Tipos

Puedes forzar un tipo específico usando corchetes antes del nombre de la variable.

# Forzar tipos específicos [int]$edad = 25 [string]$nombre = "Ana" [double]$precio = 19.99 [bool]$activo = $true [datetime]$fecha = Get-Date # Conversión de tipos $texto = "123" $numero = [int]$texto # Convierte string a entero $texto2 = [string]456 # Convierte entero a string
Atención: La conversión fallará si el valor no es compatible. Usa Try-Catch para manejar errores.

Trabajando con Arrays

Acceder a elementos

$frutas = @("Manzana", "Banana", "Naranja", "Uva") # Elemento específico (índice 0) $frutas[0] # "Manzana" # Último elemento $frutas[-1] # "Uva" # Rango de elementos $frutas[1..3] # "Banana", "Naranja", "Uva"

Modificar arrays

# Agregar elementos $frutas += "Kiwi" $frutas = $frutas + "Mango" # Agregar al principio $frutas = "Fresa", $frutas # Insertar en posición específica $array = 1, 2, 3, 4 $array[2] = 99 # Cambia posición 2

Operaciones con arrays

# Contar elementos $frutas.Count $frutas.Length # Buscar elemento $frutas -contains "Banana" # $true $frutas.IndexOf("Naranja") # 2 # Filtrar $frutas | Where-Object {$_.Length -gt 5}

Trabajando con Hashtables

Crear y acceder

# Crear hashtable $config = @{ Servidor = "localhost" Puerto = 5432 Usuario = "admin" Activo = $true } # Acceder a valores $config["Servidor"] $config.Puerto # Agregar nuevo valor $config["BaseDatos"] = "MiBaseDatos" # Modificar valor $config["Puerto"] = 3306

Operaciones con hashtables

# Obtener todas las claves $config.Keys # Obtener todos los valores $config.Values # Verificar si existe una clave $config.ContainsKey("Usuario") # $true # Eliminar una clave $config.Remove("Activo") # Contar elementos $config.Count

Variables Automáticas

PowerShell tiene variables predefinidas que proporcionan información útil.

$PWD
Current Working Directory - ruta del directorio actual
$HOME
Ruta al directorio personal del usuario
$PID
Process ID del proceso de PowerShell actual
$HOST
Información sobre el host de PowerShell
$LASTEXITCODE
Código de salida del último comando ejecutado
$ERROR
Colección de los últimos errores
$PSHOME
Ruta a la carpeta de instalación de PowerShell

Ejercicio Práctico

Práctica: Trabajando con Variables
  1. Crea variables para almacenar tu información personal: nombre, apellido, edad, ciudad
  2. Usa interpolación de strings para mostrar: "Hola, soy [nombre] de [ciudad], tengo [edad] años"
  3. Crea un array con 5 números y muestra el primer y último elemento
  4. Agrega 3 números más al array y muestra la nueva longitud
  5. Crea un hashtable para almacenar información de un producto (nombre, precio, stock, categoría)
  6. Modifica el precio del producto y agrega un nuevo campo "descuento"
  7. Convierte el string "123.45" a un número decimal
  8. Usa $PSVersionTable para mostrar la versión de PowerShell

Mejores Prácticas

  • Usa nombres descriptivos: $usuarioLogueado en lugar de $x
  • Sigue la convención PascalCase: $NombreUsuario, $FechaCreacion
  • Declara tipos cuando sea necesario: Para valores que deben ser de un tipo específico
  • Usa hashtables para datos relacionados: Es más organizado que múltiples variables sueltas
  • Evita variables globales: A menos que sea realmente necesario, usa variables locales
