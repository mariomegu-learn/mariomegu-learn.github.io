Módulo 14: Automatización y Tareas

Programador de Tareas de Windows

El Programador de Tareas permite ejecutar scripts de PowerShell automáticamente en momentos específicos o cuando ocurren ciertos eventos.

Usos comunes: Copias de seguridad, limpieza de logs, reportes periódicos, sincronización de datos.

Crear Tareas Programadas

Crear tarea básica

# Crear tarea programada $action = New-ScheduledTaskAction -Execute "PowerShell.exe" -Argument "-File C:\Scripts\MiScript.ps1" $trigger = New-ScheduledTaskTrigger -Daily -At 9am $principal = New-ScheduledTaskPrincipal -UserId "SYSTEM" -LogonType ServiceAccount -RunLevel Highest $settings = New-ScheduledTaskSettingsSet -StartWhenAvailable -DontStopOnIdleEnd # Registrar la tarea Register-ScheduledTask -TaskName "MiTarea" -Action $action -Trigger $trigger -Principal $principal -Settings $settings

Tareas programadas comunes

# Ejecutar cada hora $trigger = New-ScheduledTaskTrigger -Hourly # Ejecutar al inicio del sistema $trigger = New-ScheduledTaskTrigger -AtStartup # Ejecutar al inicio de sesión $trigger = New-ScheduledTaskTrigger -AtLogOn # Ejecutar semanalmente $trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Monday, Wednesday, Friday -At 2am # Ejecutar mensualmente $trigger = New-ScheduledTaskTrigger -Monthly -At 3am -DaysOfMonth 1

Administrar Tareas

# Listar todas las tareas Get-ScheduledTask # Buscar tarea específica Get-ScheduledTask -TaskName "MiTarea" # Listar tareas en una carpeta Get-ScheduledTask -TaskPath "\PowerShell\" # Obtener estado de tareas Get-ScheduledTask | Select-Object TaskName, State, LastRunTime, LastTaskResult # Iniciar tarea Start-ScheduledTask -TaskName "MiTarea" # Detener tarea Stop-ScheduledTask -TaskName "MiTarea" # Deshabilitar tarea Disable-ScheduledTask -TaskName "MiTarea" # Habilitar tarea Enable-ScheduledTask -TaskName "MiTarea" # Eliminar tarea Unregister-ScheduledTask -TaskName "MiTarea" -Confirm:$false

Modificar tarea

# Obtener tarea $tarea = Get-ScheduledTask -TaskName "MiTarea" # Modificar acción $action = New-ScheduledTaskAction -Execute "PowerShell.exe" -Argument "-File C:\Scripts\NuevoScript.ps1" $tarea | Set-ScheduledTask -Action $action # Modificar trigger $trigger = New-ScheduledTaskTrigger -Daily -At 10am $tarea | Set-ScheduledTask -Trigger $trigger

Jobs en Background

Start-Job

# Ejecutar job en background $job = Start-Job -ScriptBlock { Get-Process | Where-Object {$_.CPU -gt 100} Start-Sleep -Seconds 10 Write-Output "Job completado" } # Ver jobs en ejecución Get-Job # Obtener resultado Receive-Job -Job $job # Eliminar job Remove-Job -Job $job # Eliminar todos los jobs Get-Job | Remove-Job

Jobs con parámetros

# Pasar parámetros al job $computadoras = "Server01", "Server02", "Server03" $job = Start-Job -ArgumentList $computadoras -ScriptBlock { param($Computadoras) foreach ($comp in $Computadoras) { Test-Connection -ComputerName $comp -Count 1 -Quiet } } # Obtener resultado Receive-Job -Job $job

Background Jobs vs Scheduled Jobs

# Crear scheduled job (persistente) Register-ScheduledJob -Name "ReporteDiario" -ScriptBlock { Get-Process | Export-Csv "C:\Logs\procesos.csv" -NoTypeInformation } -Trigger (New-ScheduledJobTrigger -Daily -At 9am) # Listar scheduled jobs Get-ScheduledJob # Ejecutar scheduled job Start-ScheduledJob -Name "ReporteDiario"

Runspaces (Paralelismo)

Los runspaces permiten ejecutar código en paralelo de manera más eficiente que los jobs.

# Ejemplo básico de runspace param( [string[]]$Computadoras ) $runspace = [runspacefactory]::CreateRunspace() $runspace.Open() $jobs = @() foreach ($comp in $Computadoras) { $ps = [powershell]::Create() $ps.Runspace = $runspace $ps.AddScript("Test-Connection -ComputerName $comp -Count 1 -Quiet") | Out-Null $jobs += $ps.BeginInvoke() } # Esperar a que terminen $jobs | ForEach-Object { $_.WaitOne() } # Obtener resultados $results = $jobs | ForEach-Object { $_.AsyncResult } $runspace.Close()

Ejercicio Práctico

Práctica: Automatización
  1. Crea un script que limpie archivos temporales (.tmp) más viejos de 30 días
  2. Programa ese script para que se ejecute diariamente a las 2 AM
  3. Crea un job en background que verifique la conectividad a 3 servidores
  4. Programa una tarea que ejecute un reporte de procesos cada hora
  5. Crea un scheduled job que haga backup de archivos importantes cada noche
  6. Modifica una tarea existente para cambiar su horario de ejecución
  7. Crea un script que liste todas las tareas programadas y su estado
  8. Implementa paralelismo básico usando runspaces para verificar múltiples equipos

Mejores Prácticas

  • Usa cuentas apropiadas: SYSTEM para tareas del sistema, cuenta de servicio para aplicaciones
  • Configura registro de eventos: Para monitorear ejecuciones exitosas y fallidas
  • Prueba manualmente primero: Ejecuta el script manualmente antes de programarlo
  • Usa -WhatIf cuando sea posible: Para verificar qué haría el script
  • Implementa logging: Registra el resultado de cada ejecución
  • Maneja errores: Los jobs y tareas programadas no muestran errores en pantalla
  • Considera dependencias: Asegúrate de que servicios necesarios estén activos
