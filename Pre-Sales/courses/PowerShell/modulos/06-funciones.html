Módulo 06: Funciones y Procedimientos

¿Qué son las Funciones?

Las funciones son bloques de código reutilizables que realizan una tarea específica. Permiten organizar el código, evitar duplicación y mejorar la legibilidad.

# Función básica function Saludar { Write-Host "¡Hola! Bienvenido a PowerShell" } # Llamar a la función Saludar
Consejo: Usa verbos estándar de PowerShell en los nombres: Get, Set, New, Remove, Start, Stop, Test, etc.

Parámetros de Funciones

Parámetros básicos

function SaludarUsuario { param( [string]$Nombre ) Write-Host "¡Hola, $Nombre! Bienvenido" } # Llamar con parámetro SaludarUsuario -Nombre "Carlos" SaludarUsuario "Ana" # Posicional

Múltiples parámetros

function CalcularArea { param( [double]$Base, [double]$Altura ) $area = $Base * $Altura Write-Host "El área es: $area" return $area } # Llamar función $resultado = CalcularArea -Base 10 -Altura 5

Parámetros requeridos

function CrearUsuario { param( [Parameter(Mandatory=$true)] [string]$Nombre, [Parameter(Mandatory=$true)] [string]$Email ) Write-Host "Creando usuario: $Nombre" Write-Host "Email: $Email" } # Esto generará error si no se proporciona Nombre CrearUsuario -Nombre "Carlos" -Email "carlos@email.com"

Valores de Retorno

# Retornar un valor function Sumar { param( [int]$a, [int]$b ) return $a + $b } $suma = Sumar -a 5 -b 3 Write-Host "La suma es: $suma" # 8 # Retornar múltiples valores function Operaciones { param([int]$Numero) $suma = $Numero + 10 $resta = $Numero - 5 $multiplicacion = $Numero * 2 return $suma, $resta, $multiplicacion } $resultados = Operaciones 20 Write-Host "Suma: $($resultados[0])" Write-Host "Resta: $($resultados[1])" Write-Host "Multiplicación: $($resultados[2])"
Nota: En PowerShell, cualquier cosa que no se envíe a Write-Host se devuelve automáticamente.

Valores por Defecto

function Saludar { param( [string]$Nombre = "Invitado", [string]$Hora = "día" ) Write-Host "¡Buenos $Hora, $Nombre!" } # Llamadas Saludar # Buenos días, Invitado! Saludar -Nombre "Carlos" # Buenos días, Carlos! Saludar -Nombre "Ana" -Hora "noche" # Buenas noches, Ana!

Validación de Parámetros

Validación de rango

function EstablecerEdad { param( [ValidateRange(1, 150)] [int]$Edad ) Write-Host "Edad establecida: $Edad" } # Esto generará error si la edad está fuera de rango EstablecerEdad -Edad 25

Validación de longitud

function EstablecerPassword { param( [ValidateLength(8, 20)] [string]$Password ) Write-Host "Password establecida" } # Password debe tener entre 8 y 20 caracteres EstablecerPassword -Password "miPassword123"

Validación de conjunto

function EstablecerColor { param( [ValidateSet("Rojo", "Verde", "Azul")] [string]$Color ) Write-Host "Color: $Color" } # Solo acepta los valores especificados EstablecerColor -Color "Rojo"

Validación con expresión

function EstablecerEmail { param( [ValidatePattern('\w+@\w+\.\w+')] [string]$Email ) Write-Host "Email: $Email" } EstablecerEmail -Email "usuario@correo.com"

Ámbitos (Scopes)

PowerShell tiene diferentes ámbitos donde las variables y funciones existen.

# Ámbitos disponibles # Global - disponible en toda la sesión # Script - solo en el script actual # Local - ámbito actual # Private - solo en la función actual function FuncionExterna { $variableLocal = "Soy local" $global:variableGlobal = "Soy global" function FuncionInterna { $private:privateVar = "Soy privada" Write-Host $variableLocal # Funciona Write-Host $global:variableGlobal # Funciona } FuncionInterna }
Precaución: Usa variables globales con moderación. Pueden causar conflictos y hacer el código difícil de mantener.

Funciones Avanzadas

Soporte para pipeline

function ProcesarNumeros { param( [Parameter(ValueFromPipeline=$true)] [int]$Numero ) process { $cuadrado = $Numero * $Numero Write-Output $cuadrado } } # Usar en pipeline 1, 2, 3, 4, 5 | ProcesarNumeros # Salida: 1, 4, 9, 16, 25

Función con begin, process, end

function ContarElementos { param( [Parameter(ValueFromPipeline=$true)] [string]$Item ) begin { $contador = 0 Write-Host "Comenzando a procesar..." } process { $contador++ Write-Host "Procesando: $Item" } end { Write-Host "Total de elementos: $contador" } } "Manzana", "Banana", "Naranja" | ContarElementos

Ejercicio Práctico

Práctica: Creando Funciones
  1. Crea una función CalcularIMC que reciba peso y altura, y retorne el IMC
  2. Crea una función ConvertirTemperatura que convierta de Celsius a Fahrenheit y viceversa
  3. Crea una función GenerarPassword con parámetros para longitud y si debe incluir caracteres especiales
  4. Crea una función ValidarEmail que use ValidatePattern para verificar emails
  5. Crea una función que reciba un array de números y retorne el promedio, máximo y mínimo
  6. Crea una función con parámetros obligatorios para crear un usuario (nombre, email, teléfono)
  7. Crea una función que use ValidateSet para seleccionar entre tipos de archivo (txt, pdf, docx)
  8. Modifica una función para que funcione en pipeline

Mejores Prácticas

  • Usa verbos estándar: Get, Set, New, Remove, Test, Start, Stop, Restart
  • Nombres descriptivos: Get-UsuarioPorEmail en lugar de GetUser
  • Valida parámetros: Usa ValidateRange, ValidateSet, ValidatePattern
  • Documenta tus funciones: Agrega comentarios explicando qué hace y sus parámetros
  • Mantén funciones pequeñas: Cada función debe hacer una cosa bien
  • Usa parámetros obligatorios cuando sea necesario: Evita errores por falta de datos
