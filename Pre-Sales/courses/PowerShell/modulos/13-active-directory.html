Módulo 13: Active Directory

Requisitos Previos

Para trabajar con Active Directory, necesitas el módulo RSAT (Remote Server Administration Tools) instalado.

# Verificar si el módulo está instalado Get-Module -ListAvailable -Name "ActiveDirectory" # Instalar módulo AD (Windows Server) Install-WindowsFeature -Name RSAT-AD-PowerShell # Instalar RSAT en Windows 10/11 # Desde Configuración → Aplicaciones → Funcionalidades opcionales # Agregar "RSAT: Active Directory Domain Services"
Importante: Debes estar en un equipo unido al dominio para usar el módulo de Active Directory.

Gestión de Usuarios

Buscar usuarios

# Obtener todos los usuarios Get-ADUser -Filter * # Buscar usuario por nombre Get-ADUser -Filter "Name -like '*Carlos*'" # Obtener usuario específico Get-ADUser -Identity "jdoe" # Obtener con propiedades específicas Get-ADUser -Identity "jdoe" -Properties EmailAddress, Department, Title # Buscar usuarios inactivos (más de 90 días) $searchBase = "DC=empresa,DC=com" $inactiveDate = (Get-Date).AddDays(-90) Get-ADUser -Filter {(LastLogonDate -lt $inactiveDate) -and (Enabled -eq $true)} -Properties LastLogonDate

Crear usuario

# Crear nuevo usuario New-ADUser -Name "Juan Pérez" ` -GivenName "Juan" ` -Surname "Pérez" ` -SamAccountName "jperez" ` -UserPrincipalName "jperez@empresa.com" ` -Path "OU=Usuarios,DC=empresa,DC=com" ` -AccountPassword (ConvertTo-SecureString "Password123!" -AsPlainText -Force) ` -Enabled $true # Establecer información adicional Set-ADUser -Identity "jperez" ` -EmailAddress "jperez@empresa.com" ` -Department "IT" ` -Title "Administrador"

Modificar usuario

# Habilitar/deshabilitar usuario Enable-ADAccount -Identity "jperez" Disable-ADAccount -Identity "jperez" # Cambiar contraseña Set-ADAccountPassword -Identity "jperez" -Reset -NewPassword (ConvertTo-SecureString "NewPassword123!" -AsPlainText -Force) # Modificar atributos Set-ADUser -Identity "jperez" -Replace @{Department="Marketing"; Title="Gerente"} # Agregar y quitar valores Set-ADUser -Identity "jperez" -Add @{ProxyAddresses="SMTP:juan.perez@empresa.com"} Set-ADUser -Identity "jperez" -Remove @{ProxyAddresses="SMTP:juan.perez@empresa.com"} # Bloquear/desbloquear cuenta Unlock-ADAccount -Identity "jperez"

Eliminar usuario

# Eliminar usuario (requiere confirmación) Remove-ADUser -Identity "jperez" # Eliminar sin confirmar Remove-ADUser -Identity "jperez" -Confirm:$false # Eliminar y mover a contenedor eliminados Remove-ADUser -Identity "jperez" -Recursive

Gestión de Grupos

Buscar grupos

# Obtener todos los grupos Get-ADGroup -Filter * # Buscar grupo específico Get-ADGroup -Identity "Administradores" # Buscar grupos por nombre Get-ADGroup -Filter "Name -like '*Admin*'" # Obtener miembros de un grupo Get-ADGroupMember -Identity "Administradores" # Obtener grupos de los que es miembro un usuario Get-ADPrincipalGroupMembership -Identity "jperez"

Crear grupo

# Crear grupo New-ADGroup -Name "Desarrolladores" ` -GroupScope Global ` -GroupCategory Security ` -Path "OU=Grupos,DC=empresa,DC=com" # Grupo de distribución (no de seguridad) New-ADGroup -Name "Boletin" ` -GroupScope Universal ` -GroupCategory Distribution ` -Path "OU=Grupos,DC=empresa,DC=com"

Administrar miembros

# Agregar miembro al grupo Add-ADGroupMember -Identity "Desarrolladores" -Members "jperez", "mgonzalez" # Quitar miembro Remove-ADGroupMember -Identity "Desarrolladores" -Members "jperez" -Confirm:$false # Reemplazar todos los miembros Set-ADGroup -Identity "Desarrolladores" -Replace @{member="CN=Ana,CN=Users,DC=empresa,DC=com;CN=Carlos,CN=Users,DC=empresa,DC=com"} # Verificar si un usuario es miembro Get-ADGroupMember -Identity "Desarrolladores" | Where-Object {$_.SamAccountName -eq "jperez"}

Unidades Organizacionales (OU)

# Obtener todas las OU Get-ADOrganizationalUnit -Filter * # Obtener OU específica Get-ADOrganizationalUnit -Identity "OU=Usuarios,DC=empresa,DC=com" # Crear OU New-ADOrganizationalUnit -Name "Departamentos" -Path "DC=empresa,DC=com" # Mover objeto a OU Move-ADObject -Identity "CN=Juan,CN=Users,DC=empresa,DC=com" -TargetPath "OU=Usuarios,DC=empresa,DC=com" # Obtener objetos en una OU Get-ADUser -Filter * -SearchBase "OU=Usuarios,DC=empresa,DC=com" # Buscar en subcarpetas Get-ADUser -Filter * -SearchBase "DC=empresa,DC=com" -SearchScope Subtree

Exportar Datos

# Exportar usuarios a CSV Get-ADUser -Filter * -Properties EmailAddress, Department, Title | Select-Object Name, SamAccountName, EmailAddress, Department, Title, Enabled | Export-Csv -Path "C:\Reportes\usuarios.csv" -NoTypeInformation -Encoding UTF8 # Exportar miembros de grupos Get-ADGroup -Filter * | ForEach-Object { $group = $_ $members = Get-ADGroupMember -Identity $group | Select-Object -ExpandProperty SamAccountName [PSCustomObject]@{ Grupo = $group.Name Miembros = ($members -join ";") } } | Export-Csv -Path "C:\Reportes\grupos.csv" -NoTypeInformation # Exportar estructura de OU Get-ADOrganizationalUnit -Filter * | Select-Object DistinguishedName, Name | Export-Csv -Path "C:\Reportes\ou.csv" -NoTypeInformation

Ejercicio Práctico

Práctica: Active Directory
  1. Verifica si el módulo ActiveDirectory está instalado
  2. Obtén información de tu usuario actual: Get-ADUser -Identity $env:USERNAME -Properties *
  3. Lista todos los usuarios del dominio
  4. Obtén los grupos de los que eres miembro
  5. Busca usuarios inactivos (más de 60 días)
  6. Crea un script que exporte todos los usuarios a CSV con sus propiedades principales
  7. Busca todos los usuarios de un departamento específico
  8. Obtén el tamaño de cada grupo (número de miembros)
  9. Crea un reporte de cuentas deshabilitadas

Mejores Prácticas

  • Usa filtros específicos: Evita obtener todos los objetos si no es necesario
  • Prueba con -WhatIf: Antes de eliminar o modificar objetos críticos
  • Exporta antes de modificar: Ten un backup de los datos originales
  • Usa transacciones: Para operaciones que deben completarse juntas
  • Documenta cambios: Mantén un log de modificaciones importantes
  • Usa grupos anidados: Para simplificar la administración
