Módulo 11: Remoting y WS-Management

¿Qué es PowerShell Remoting?

PowerShell Remoting permite ejecutar comandos en computadoras remotas. Utiliza WS-Management (WinRM) como protocolo de transporte, permitiendo administrar servidores y equipos de forma centralizada.

Importante: PowerShell Remoting es diferente de RDP. Ejecutas comandos, no ves la interfaz gráfica.

Configurar Remoting

En el servidor/host remoto

# Habilitar PowerShell Remoting (requiere admin) Enable-PSRemoting -Force # Esto configura: # - Servicio WinRM # - Firewall para tráfico WinRM # - Listener HTTP en puerto 5985 # Verificar estado Get-Service WinRM

Configuración manual

# Configurar WinRM manualmente winrm quickconfig # Crear listener winrm create winrm/config/Listener?Address=*+Transport=HTTP # Configurar firewall New-NetFirewallRule -Name "WinRM-HTTP" -DisplayName "WinRM HTTP" -Enabled True -Profile Any -LocalPort 5985 -Protocol TCP

Invoke-Command

Invoke-Command ejecuta comandos en computadoras remotas.

Sintaxis básica

# Ejecutar comando remoto simple Invoke-Command -ComputerName "Server01" -ScriptBlock { Get-Process } # Ejecutar en múltiples equipos Invoke-Command -ComputerName "Server01", "Server02", "Server03" -ScriptBlock { Get-Service } # Ejecutar comando con parámetros Invoke-Command -ComputerName "Server01" -ScriptBlock { param($NombreProceso) Get-Process -Name $NombreProceso } -ArgumentList "chrome"

Usar credenciales

# Pedir credenciales $cred = Get-Credential # Ejecutar con credenciales Invoke-Command -ComputerName "Server01" -Credential $cred -ScriptBlock { Get-Process | Select-Object -First 10 }

Ejecutar script remoto

# Ejecutar un script local en equipo remoto Invoke-Command -ComputerName "Server01" -FilePath "C:\Scripts\MiScript.ps1" # Pasar parámetros al script Invoke-Command -ComputerName "Server01" -FilePath "C:\Scripts\MiScript.ps1" -ArgumentList "param1", "param2"

Sesiones Remotas

Crear sesión

# Crear sesión remota $session = New-PSSession -ComputerName "Server01" # Crear múltiples sesiones $sessions = New-PSSession -ComputerName "Server01", "Server02", "Server03" # Crear con credenciales $session = New-PSSession -ComputerName "Server01" -Credential $cred

Usar sesión

# Ejecutar comando en sesión existente Invoke-Command -Session $session -ScriptBlock { Get-Process } # Ejecutar en todas las sesiones Invoke-Command -Session $sessions -ScriptBlock { Get-Service | Where-Object {$_.Status -eq "Running"} }

Administrar sesiones

# Listar sesiones Get-PSSession # Eliminar sesión específica Remove-PSSession -Session $session # Eliminar todas las sesiones Get-PSSession | Remove-PSSession # Conectarse a una sesión (interactivo) Enter-PSSession -ComputerName "Server01" # Salir de sesión interactiva Exit-PSSession

Transferir Archivos

Copiar al equipo remoto

# Copiar archivo al equipo remoto Copy-Item -Path "C:\Local\archivo.txt" -Destination "\\Server01\C$\Temp\" # Usar Copy-Item con sesión $session = New-PSSession -ComputerName "Server01" Copy-Item -Path "C:\Local\archivo.txt" -Destination "C:\Temp\" -ToSession $session

Copiar desde equipo remoto

# Copiar archivo desde remoto Copy-Item -Path "\\Server01\C$\Temp\archivo.txt" -Destination "C:\Local\" # Usar Copy-Item desde sesión Copy-Item -Path "C:\Temp\archivo.txt" -Destination "C:\Local\" -FromSession $session

Configuración de WinRM

Ver configuración

# Ver configuración de WinRM winrm enumerate winrm/config # Ver listeners winrm enumerate winrm/config/listener # Ver configuración del cliente winrm enumerate winrm/config/client

Configurar trusted hosts

# Ver hosts confiables winrm get winrm/config/client # Agregar hosts confiables winrm set winrm/config/client '@{TrustedHosts="Server01,Server02"}' # Permitir todos los hosts (NO recomendado para producción) winrm set winrm/config/client '@{TrustedHosts="*"}'

Seguridad en Remoting

UseSSL

# Configurar listener HTTPS winrm create winrm/config/Listener?Address=*+Transport=HTTPS # Conectar usando SSL Invoke-Command -ComputerName "Server01" -UseSSL -ScriptBlock { Get-Process }

Credenciales delegadas

# Configurar CredSSP (para doble salto) Enable-WSManCredSSP -Role Client -DelegateComputer "Server01" Enable-WSManCredSSP -Role Server # Usar CredSSP Invoke-Command -ComputerName "Server01" -Authentication CredSSP -Credential $cred -ScriptBlock { # Puede acceder a recursos de red Get-SmbShare }

Ejercicio Práctico

Práctica: PowerShell Remoting
  1. Verifica si WinRM está ejecutándose en tu equipo: Get-Service WinRM
  2. Habilita PowerShell Remoting: Enable-PSRemoting -Force
  3. Crea una sesión remota a un equipo de tu red (o localhost para pruebas)
  4. Ejecuta Get-Process en un equipo remoto usando Invoke-Command
  5. Crea múltiples sesiones a diferentes equipos
  6. Obtén información de servicios en todos los equipos simultáneamente
  7. Copia un archivo a un equipo remoto usando Copy-Item con sesión
  8. Prueba Enter-PSSession para una sesión interactiva
  9. Configura TrustedHosts para tu entorno de pruebas

Mejores Prácticas

  • Usa UseSSL en producción: Para cifrar la comunicación
  • Limita TrustedHosts: Solo agregar equipos necesarios
  • Usa sesiones para múltiples comandos: Más eficiente que crear nuevas conexiones
  • Limpia sesiones: Siempre elimina sesiones que ya no uses
  • Usa credenciales apropiadas: Mínimo privilegios necesarios
  • Considera CredSSP para recursos de red: Cuando necesitas acceso a terceros
