Módulo 07: Manejo de Errores

¿Por qué Manejar Errores?

El manejo de errores es esencial para crear scripts robustos y confiables. Permite que tu script continúe funcionando incluso cuando algo sale mal, y proporciona información clara sobre qué sucedió.

Importante: Un buen manejo de errores hace que tus scripts sean más profesionales y fáciles de depurar.

Try-Catch-Finally

La estructura Try-Catch-Finally es la principal forma de manejar errores en PowerShell.

# Estructura básica try { # Código que puede generar error $resultado = 10 / 0 Write-Host "Resultado: $resultado" } catch { # Se ejecuta si hay un error Write-Host "¡Ocurrió un error!" Write-Host "Mensaje: $($_.Exception.Message)" } finally { # Siempre se ejecuta, haya error o no Write-Host "Ejecución finalizada" }

Ejemplo práctico

try { $archivo = Get-Content "archivo_no_existe.txt" Write-Host "Archivo leído correctamente" } catch { Write-Host "No se pudo leer el archivo" Write-Host "Tipo de error: $($_.Exception.GetType().FullName)" Write-Host "Mensaje: $($_.Exception.Message)" } finally { Write-Host "Operación completada" }

Catch Múltiple

Puedes tener múltiples bloques catch para manejar diferentes tipos de errores.

try { $numero = [int]Read-Host "Ingresa un número" Write-Host "Número ingresado: $numero" } catch [FormatException] { Write-Host "Error: El texto no es un número válido" } catch [OverflowException] { Write-Host "Error: El número es demasiado grande" } catch { Write-Host "Error inesperado: $($_.Exception.Message)" }

Manejo de errores de archivo

try { $contenido = Get-Content "ruta\archivo.txt" } catch [System.IO.FileNotFoundException] { Write-Host "El archivo no existe" } catch [System.IO.DirectoryNotFoundException] { Write-Host "El directorio no existe" } catch [System.UnauthorizedAccessException] { Write-Host "No tienes permisos para leer el archivo" } catch { Write-Host "Error general: $($_.Exception.Message)" }

Write-Error

Write-Error genera errores que pueden ser capturados por bloques catch.

# Generar un error personalizado function ValidarEdad { param([int]$Edad) if ($Edad -lt 0 -or $Edad -gt 150) { Write-Error "La edad debe estar entre 0 y 150" -Category InvalidArgument } Write-Host "Edad válida: $Edad" } # Capturar el error try { ValidarEdad -Edad 200 } catch { Write-Host "Error capturado: $($_.Exception.Message)" }

Categorías de error

Categoría Descripción
NotSpecified Error no especificado
InvalidArgument Argumento inválido
InvalidOperation Operación inválida
ResourceUnavailable Recurso no disponible
PermissionDenied Permiso denegado

Throw

Throw genera un error que detiene la ejecución y puede ser capturado por un try-catch externo.

function CrearUsuario { param($Nombre, $Email) if ([string]::IsNullOrWhiteSpace($Nombre)) { throw "El nombre es obligatorio" } if ($Email -notmatch '\w+@\w+\.\w+') { throw "El email no es válido" } Write-Host "Usuario creado: $Nombre" } # Usar throw try { CrearUsuario -Nombre "" -Email "test@email.com" } catch { Write-Host "Error: $($_.Exception.Message)" }
Diferencia: Write-Error genera un error no terminante. Throw genera un error terminante que detiene la ejecución.

$Error y $ErrorActionPreference

Variable $Error

# Ver el último error $Error[0] $Error[0].Exception.Message # Ver todos los errores $Error | ForEach-Object { $_.Exception.Message } # Limpiar errores $Error.Clear() # Ver tipo de error $Error[0].Exception.GetType().FullName

$ErrorActionPreference

# Configurar comportamiento global $ErrorActionPreference = "Stop" # Detener en cualquier error $ErrorActionPreference = "Continue" # Continuar aunque haya errores $ErrorActionPreference = "SilentlyContinue" # Ignorar errores # Configurar para un comando específico Get-Content "archivo.txt" -ErrorAction Stop Get-Content "archivo.txt" -ErrorAction SilentlyContinue

Logging de Errores

function LogError { param( [string]$Mensaje, [string]$ArchivoLog = "C:\Logs\error.log" ) $timestamp = Get-Date -Format "yyyy-MM-dd HH:mm:ss" $logEntry = "[$timestamp] ERROR: $Mensaje" # Crear directorio si no existe $logPath = Split-Path $ArchivoLog if (!(Test-Path $logPath)) { New-Item -ItemType Directory -Path $logPath -Force } # Escribir en log Add-Content -Path $ArchivoLog -Value $logEntry } # Uso try { $resultado = 10 / 0 } catch { LogError -Mensaje "Error al dividir: $($_.Exception.Message)" }

Ejercicio Práctico

Práctica: Manejo de Errores
  1. Crea una función que divida dos números y maneje el error de división por cero
  2. Crea una función que lea un archivo y maneje errores de archivo no encontrado y permisos
  3. Implementa múltiples catch para diferentes tipos de errores de conversión de tipos
  4. Crea una función que use throw para validar parámetros obligatorios
  5. Implementa logging de errores a un archivo con timestamp
  6. Crea un script que use $ErrorActionPreference = "Stop" y capture errores
  7. Usa Write-Error con diferentes categorías de error
  8. Crea un bloque try-catch-finally que siempre limpie recursos en finally

Mejores Prácticas

  • Sé específico en catch: Captura tipos de error específicos cuando sea posible
  • Usa Write-Error para advertencias: Y throw para errores que deben detener la ejecución
  • Siempre registra errores: Usa logging para poder depurar problemas en producción
  • Limpia recursos en Finally: Cierra archivos, conexiones, etc.
  • No ignores errores: Evita usar -ErrorAction SilentlyContinue sin razón
  • Proporciona mensajes claros: Los mensajes de error deben ser útiles para el usuario
