Módulo 02: El Entorno de PowerShell

Entornos de PowerShell

PowerShell puede ejecutarse en diferentes entornos, cada uno con sus propias ventajas según el tipo de trabajo que realices.

1. Windows Terminal (Recomendado)

La terminal moderna de Microsoft que soporta múltiples pestañas, perfiles y personalización avanzada.

# Instalar Windows Terminal desde Microsoft Store # O usar winget winget install Microsoft.WindowsTerminal

2. PowerShell ISE (Integrated Scripting Environment)

Entorno de scripting integrado incluido con Windows. Ideal para desarrollar scripts en Windows, aunque ya no recibe nuevas características.

3. Visual Studio Code (Recomendado para desarrollo)

Editor de código con extensión oficial de PowerShell. Ofrece debugging, IntelliSense y control de versiones integrado.

Recomendación: Usa VS Code para desarrollar scripts y Windows Terminal para ejecutar comandos interactivamente.

Personalización del Prompt

El prompt es la línea que muestra antes de cada comando. Puedes personalizarlo para mostrar información útil.

Prompt básico personalizado

# Función para personalizar el prompt function prompt { $hora = Get-Date -Format "HH:mm" $ruta = (Get-Location).Path "$hora PS $ruta> " }

Prompt con información de git

function prompt { $branch = git branch --show-current 2>$null $ruta = Split-Path (Get-Location).Path -Leaf if ($branch) { "PS $ruta [$branch]> " } else { "PS $ruta> " } }

El Perfil de PowerShell

El perfil es un script que se ejecuta cada vez que abres PowerShell. Perfecto para cargar funciones, aliases y configuraciones.

Ubicación del perfil

# Ver la ruta del perfil $PROFILE # Ejemplo de ruta # C:\Users\Usuario\Documents\PowerShell\Microsoft.PowerShell_profile.ps1

Crear un perfil

# Crear el archivo de perfil si no existe if (!(Test-Path -Path $PROFILE)) { New-Item -ItemType File -Path $PROFILE -Force } # Abrir el perfil en VS Code code $PROFILE

Contenido típico del perfil

# Alias personalizados Set-Alias v code Set-Alias ll Get-ChildItem # Funciones útiles function gs { git status } function gp { git push } function gc { git commit -m $args[0] } # Importar módulos Import-Module PSReadLine # Configuración de PSReadLine Set-PSReadLineOption -PredictionSource History Set-PSReadLineOption -Colors @{ Command = 'Cyan' }

Políticas de Ejecución

PowerShell tiene políticas de seguridad que controlan qué scripts pueden ejecutarse.

Ver la política actual

Get-ExecutionPolicy -List

Niveles de política

Política Descripción
Restricted No permite scripts (por defecto)
AllSigned Solo scripts firmados por editor confiable
RemoteSigned Scripts locales sí, remotos deben estar firmados (Recomendado)
Unrestricted Todos los scripts sin advertencia
Bypass Todo sin advertencias ni bloqueos

Cambiar la política de ejecución

# Cambiar para el usuario actual (requiere admin) Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser # Cambiar solo para la sesión actual Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
Seguridad: No uses Unrestricted o Bypass en producción. RemoteSigned es el equilibrio recomendado.

Comandos de Entorno

$PSVersionTable
Muestra información detallada sobre la versión de PowerShell
$Host
Objeto que representa el host de PowerShell actual
$PID
Process ID del proceso de PowerShell actual
$PWD
Current Working Directory (directorios actual)
$HOME
Ruta a la carpeta personal del usuario
$PSScriptRoot
Directorio donde se encuentra el script actual (solo en scripts)

Ejercicio Práctico

Práctica: Configurando tu Entorno
  1. Verifica tu versión de PowerShell: $PSVersionTable.PSVersion
  2. Comprueba la política de ejecución: Get-ExecutionPolicy -List
  3. Si es necesario, cambia la política: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  4. Verifica la ruta de tu perfil: $PROFILE
  5. Crea tu perfil si no existe: New-Item -ItemType File -Path $PROFILE -Force
  6. Agrega al menos 3 aliases útiles a tu perfil
  7. Agrega una función personalizada que te sea útil
  8. Reinicia PowerShell y verifica que tu configuración se aplicó
  9. Personaliza el prompt para mostrar la hora actual

Mejores Prácticas

  • Usa Windows Terminal o VS Code: Son los entornos más modernos y con mejor soporte
  • Mantén tu perfil organizado: No sobrecargues tu perfil con demasiadas funciones
  • Usa políticas de ejecución seguras: RemoteSigned es el mejor equilibrio
  • Personaliza con moderación: Solo agrega lo que realmente uses
  • Versiona tu perfil: Guarda tu perfil en un repositorio Git para tenerlo en cualquier equipo
