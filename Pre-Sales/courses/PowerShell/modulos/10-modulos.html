Módulo 10: Módulos y Snap-ins

¿Qué son los Módulos?

Los módulos son paquetes que contienen cmdlets, funciones, providers, scripts y otros recursos de PowerShell. Permiten organizar y reutilizar código de manera eficiente.

Ventaja: Los módulos permiten compartir código, mantener versiones y organizar funcionalidades relacionadas.

Gestionar Módulos

Buscar módulos

# Buscar en PowerShell Gallery Find-Module -Name "*azure*" # Buscar módulos específicos Find-Module -Name "AzureAD" Find-Module -Name "PSScriptAnalyzer"

Instalar módulos

# Instalar un módulo Install-Module -Name "Az" # Instalar sin confirmar Install-Module -Name "Az" -Force # Instalar para todos los usuarios Install-Module -Name "ModuleName" -Scope AllUsers # Instalar versión específica Install-Module -Name "ModuleName" -RequiredVersion "1.0.0"

Ver módulos instalados

# Listar módulos instalados Get-Module -ListAvailable # Listar módulos cargados Get-Module # Buscar módulo específico Get-Module -ListAvailable -Name "Az*"

Actualizar módulos

# Actualizar PowerShellGet Update-Module -Name "PowerShellGet" # Actualizar un módulo específico Update-Module -Name "Az" # Actualizar todos los módulos Get-Module -ListAvailable | Update-Module -Force

Eliminar módulos

# Desinstalar módulo Uninstall-Module -Name "ModuleName" # Desinstalar versión específica Uninstall-Module -Name "ModuleName" -RequiredVersion "1.0.0"

Importar y Exportar Módulos

Importar módulos

# Importar módulo Import-Module -Name "Az" # Importar con versión específica Import-Module -Name "Az" -RequiredVersion "2.0.0" # Importar sin cargar dependencias Import-Module -Name "ModuleName" -PassThru # Verificar si un módulo está importado Get-Module -Name "Az"

Exportar funciones

# Ver funciones exportadas por un módulo Get-Module -Name "Az" | Select-Object -ExpandProperty ExportedCommands # Obtener cmdlets de un módulo Get-Command -Module "Az"

Crear tu Propio Módulo

Estructura básica

Un módulo consta de al menos un archivo .psm1 (script module) y opcionalmente un archivo .psd1 (manifest).

Crear archivo de módulo (.psm1)

# MiModulo.psm1 # Función 1 function Get-Hello { param( [string]$Nombre = "Mundo" ) Write-Host "¡Hola, $Nombre!" } # Función 2 function Get-FechaActual { Get-Date -Format "yyyy-MM-dd HH:mm:ss" } # Función 3 function Test-Conexion { param([string]$Computadora = "localhost") Test-Connection -ComputerName $Computadora -Count 1 -Quiet }

Crear manifiesto del módulo (.psd1)

# Nueva-ModuleManifest -Path "MiModulo.psd1" -ScriptModuleFile "MiModulo.psm1" # Contenido del manifiesto @{ RootModule = 'MiModulo.psm1' ModuleVersion = '1.0' GUID = 'a1b2c3d4-e5f6-7890-abcd-ef1234567890' Author = 'Tu Nombre' CompanyName = 'Tu Empresa' Copyright = '© 2024 Tu Empresa. Todos los derechos reservados.' Description = 'Mi módulo personalizado de PowerShell' PowerShellVersion = '5.1' FunctionsToExport = @('Get-Hello', 'Get-FechaActual', 'Test-Conexion') CmdletsToExport = @() VariablesToExport = @() AliasesToExport = @() }

Instalar tu módulo

# Copiar a la carpeta de módulos $modulePath = "$env:USERPROFILE\Documents\PowerShell\Modules" Copy-Item -Path "MiModulo" -Destination $modulePath -Recurse # Importar y usar Import-Module -Name "MiModulo" Get-Hello -Nombre "Carlos"

PowerShell Gallery

PowerShell Gallery es el repositorio central de módulos y scripts de PowerShell.

# Ver información de un módulo Find-Module -Name "Az" | Select-Object Name, Version, Description, Author # Descargar módulo sin instalar Save-Module -Name "Az" -Path "C:\Downloads" # Buscar por etiqueta Find-Module -Tag "Azure" # Buscar módulos con actualizaciones Get-Module -ListAvailable | Where-Object { $_.Version -lt (Find-Module -Name $_.Name).Version }

Módulos Útiles Recomendados

PSScriptAnalyzer
Analizador de código para verificar mejores prácticas
PlatyPS
Genera documentación para módulos desde markdown
Pester
Framework de testing para PowerShell
Posh-Git
Extensión de Git para PowerShell con información del repositorio
PSWriteHTML
Crea reportes HTML desde PowerShell

Ejercicio Práctico

Práctica: Trabajando con Módulos
  1. Busca e instala el módulo "PSScriptAnalyzer"
  2. Verifica qué módulos tienes instalados con Get-Module -ListAvailable
  3. Importa el módulo "Az.Accounts" y verifica su versión
  4. Crea tu propio módulo con al menos 3 funciones útiles
  5. Crea el manifiesto .psd1 para tu módulo
  6. Instala tu módulo en la carpeta de módulos de PowerShell
  7. Importa tu módulo y prueba todas las funciones
  8. Busca un módulo en PowerShell Gallery que te sea útil e instálalo
  9. Actualiza todos tus módulos a la última versión

Mejores Prácticas

  • Usa nombres descriptivos: Para tu módulo y funciones
  • Documenta tu módulo: Incluye descripción y comentarios
  • Usa versiones: Controla cambios con versionamiento semántico
  • Exporta solo lo necesario: No expongas funciones internas
  • Mantén módulos actualizados: Revisa actualizaciones regularmente
  • Usa PowerShell Gallery: Es la fuente oficial y segura
