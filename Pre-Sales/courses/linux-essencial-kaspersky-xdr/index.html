Módulo 0

Bienvenida al curso

Capacitación técnica-práctica para implementar y administrar KSC Linux, KES for Linux y su integración conceptual con EDR/XDR Optimum.

Objetivos

  • Entender roles de Linux, KSC, KESL y EDR/XDR.
  • Practicar comandos seguros.
  • Recolectar evidencias y ejecutar checklists.

Audiencia y prerrequisitos

Orientado a ingenieros de implementación, preventa, soporte N1/N2, administradores Linux y consultores técnicos. Se requiere manejo básico de terminal, redes TCP/IP, DNS y permisos sudo.

Diferencias clave: Linux puede ser plataforma de KSC Linux o endpoint protegido por KESL. KSC centraliza administración. KESL provee protección EPP Linux. EDR/XDR Optimum agrega detección, contexto y respuesta cuando licencia, versiones y componentes son compatibles.

Advertencia: Este curso es educativo y operativo. Antes de ejecutar procedimientos en producción, valide la documentación oficial vigente de Kaspersky.

1. ¿Qué comando permite ver la distribución Linux instalada?

El archivo /etc/os-release muestra información de la distribución Linux.

2. ¿Qué comando valida el FQDN?

hostname -f devuelve el nombre completamente calificado si está configurado.

3. ¿Qué utilidad se asocia a backup de KSC Linux?

klbackup se usa para respaldar/restaurar datos de KSC Linux según documentación oficial.

4. ¿Qué herramienta administra KESL desde CLI?

kesl-control expone comandos de ayuda, tareas, licencias y cuarentena.

5. ¿Qué se debe hacer antes de cambios críticos?

Los cambios críticos requieren respaldo, aprobación y plan de reversa.
